******** VERSION 2.0.0 ********
Fluvio:
- All of the code has been rewritten, and the object model has been simplified. Performance tests have shown Fluvio 2 to run anywhere between 2x-4x faster, use less memory, and create less garbage.
- All shaders have been rewritten into surface shaders for more unified support across platforms. Shader-related effects have all been combined into a single FluidEffect script, which should be applied on a camera.
- Shaders now support lighting, normal maps, specular highlights, reflective cubemaps, and shadows. Shader level of detail and fallback shaders are also available.
- Added a simple decal system with support for object pooling and prefab creation
- FluidBillboards have been replaced. Billboards are now rendered either using Shuriken (Unity 3.5+), or the Legacy Particle Emitter.
- FluidParticlePlugins have been removed (Prefabs can still be spawned along with particles, though). All ParticlePlugin functionality can be handled with OnFluidCollision____ events.
- Fluid audio components offer much better performance.
- An AdvancedFluidSettings component has been added, exposing some performance-related API functions.
- Multithreading can be toggled at runtime.
- Added fluid platform profiles - fluid settings can be changed at start time for different devices or resolutions.
- Editor support has been added for saving and loading fluids.

Bug Fixes:
- Fixed a bug with fluid particles not always emitting when they should.

Example Project:
- Fluvio 2 ships with a brand new example project, with new demo scenes!
	* Paint - A demo showcasing the new shaders and the decal system
	* Water Wheel - A demo showcasing a water-like fluid and physical interactions
	* Gravity - A stress-test demo with a massive particle count and a gravity solver
	* Water Wheel (Mobile) - A 2D version of Water Wheel, using the mobile shaders.
- Updated and cleaned up the example project scripts
- Quality settings can now be changed at runtime

Roadmap:
- Fluid splash effects (separated from decals) are coming soon!
- A FluidVoxelEmitter and FluidMeshEmitter rewrite (for more stability)
- A softbody rewrite and implementation
- An animation system rewrite and implementation

******** VERSION 1.2 ********
NOTE: Many files have been renamed (all Unity component files, for example). PLEASE back up your project before upgrading!

File/Folder Structure Changes:
- Renamed IFluvio to IFluvioObject
- Renamed Fluvio.cs to FluvioBase.cs
- Added namespaces (where possible!)
- Added the _Threading classes and folder
- Added the KDTree class
- Added the FluidAnimation classes and folders
- Added sample animations
- Added the FluvioTools, FluvioEditor, FluvioExtensions classes

Fluvio:
- Added some more collision events, for fluid particle plugins:
	* public override void OnFluidAnyCollisionEnter()
	* public override void OnFluidAnyCollisionStay()
	* public override void OnFluidAnyCollisionExit()

- Fluvio Tools Window: Added a new window with common tools accessable at Window > Fluvio > Fluvio Tools.
- Fluid Animation: Fluids can now be pre-baked and converted into animations. This greatly increases performance, while sacrificing interactivity.
- New nearest neighbor algorithm: A KD-Tree algorithm has been implemented for nearest neighbor particle search, greatly increasing overall physics performance.
- The frequency of tree updates can be adjusted (check the documentation for Fluid).
- Multithreading support: Fluvio now supports multithreading. This is enabled by default and can be turned off with the #MULTITHREADING define in Fluid.cs.
- More documentation: The documentation has been updated with a few new API functions and a Getting Started Guide.
- Billboard meshes now generate normals and tangents.

Example Project:
- Created new example scenes for fluid animation.
- Fixed some bugs and inconsistencies.
- The demo has been split into two example scenes. The installation instructions for the example project have also changed slightly.

Issues:
- Animations currently do not record smoothing distance changes. This is planned for a future release.
- Animations will playback with all particles belonging to one emitter - this is the result of a bug. A fix is planned for a future release.
- Multithreading and animation are very new, and while they have been found to be stable, bugs may arise.
- Better softbodies are *finally* coming soon! =)

******** VERSION 1.1.5 ********
File/Folder Structure Changes:
- Renamed FluidParticleInfo to FluidParticlePhysicsInfo
- Documentation has moved to a seperate folder

Fluvio:
- Fixed Unity 3.4 warnings. Fluvio currently requires 3.4 but that requirement may go away in the future.
- Collision events!

** Usable on all GameObjects:
	* void OnFluidCollisionEnter(FluidCollision other)
	* void OnFluidCollisionStay(FluidCollision other)
	* void OnFluidCollisionExit(FluidCollision other)

** Usable on fluid particle plugins only:
	
	* public override void OnFluidSelfCollisionEnter()
	* public override void OnFluidSelfCollisionStay()
	* public override void OnFluidSelfCollisionExit()
	* public override void OnFluidCollisionEnd()
	
See the documentation for more details. Sending event messages can be toggled on and off in the fluid properties.

- Updated Fluid Sound to work with collision events. Sending event messages are required in order to use the Fluid Audio Manager.
- Renamed FluidParticleInfo to FluidParticlePhysicsInfo and changed it from a class into a struct
- Created interface type FluidCollision
- Created interface type FluidSolver
