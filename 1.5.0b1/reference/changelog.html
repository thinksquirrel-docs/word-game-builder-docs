Changelog

Fluvio 2.2.2

New Features

Mobile and No GrabPass Shaders

Some new shaders have been added, offering a wide range of compatibility. They can be found under the following categories:

  • Fluvio/No GrabPass
  • Fluvio/Mobile

Both categories of shaders use a static texture for refraction and are fully supported with Unity Free. The mobile shaders are optimized even more. It is recommended to use these shaders on mobile platforms.

Changes and Fixes

Other Changes

  • (Unity 4 only) Tweaked the example project to not throw any errors in the free edition of Unity (removed extra image effects and other Pro-only scripts).
  • (Unity 4 only) Re-baked lightmaps in the example project and disabled HDR rendering



Fluvio 2.2.1

New Features

Streamlined Installation

Both Unity 3.5 and Unity 4 users can install the latest version of Fluvio direclty from the Asset Store. The correct version of Fluvio will be automatically installed based on the Unity Editor's version.

Thinksquirrel Product Registration

You can now optionally register your copy of Fluvio. This ensures full product support - you can also sign up for an email notification whenever a new update is released.

You can visit the registration page here.

Changes and Fixes

Other Changes

  • Fixed a bug where in-editor documentation would crash Unity.
  • Added an intensity value to the Refractive/Multiply and Refractive/VertexLit Blended shaders.
  • Fixed a regression where default fluids would not initialize their distortion texture properly.
  • Vertex lighting now works correctly for lit shaders.
  • Fixed a depreciated code warning in the Fluid class.
  • Fixed a regression/crash bug when using multiple legacy fluid effects.
  • Moved the Thinksquirrel > Fluvio menu to GameObject > Fluvio and Window > Fluvio to comply with new Asset Store guidelines.

Compatibility Notice

Fluvio now uses Shuriken for particle rendering and requires Unity 3.5+. The legacy particle system is no longer supported. Shuriken world collision requires Unity 4.0+. Fluvio 2.2 is not backwards compatible with older Fluvio versions - please remove older versions of Fluvio from any existing project before upgrading!



Fluvio 2.2.0

New Features

Better Performance

Performance, performance, performance! The solver has been rewritten from the ground up. Particles are no longer by default tied to game objects and rigidbodies, greatly reducing overhead.

New Shaders

Brand new fluid shaders, with refraction! Refraction uses a grab pass and requires Unity Pro. No more Fluid Effect component required!

Full Shuriken Compatibility

Fluids will now work with all Shuriken settings, gradients and curves (except for the Emission/Shape modules and start speed - these are handled by Fluvio). Advanced features like collision and sub-emitters are supported.

Fluid Attractors

Introducing fluid attractors! Attractors will pull nearby fluid particles to fill their volume. Fluid attractors are force-based and respond to external stimuli. Fluids can fill all sorts of volumes, including custom meshes.

New Examples

Brand new example scenes, including a new Unity 4 high quality example scene!

Fluid Templates

Introducing fluid templates! Most Fluvio components can be serialized to and from JSON-based fluid templates. This works at both runtime and from within the editor. NOTE: Object references are currently not serialized, and templates are saved on a per-component basis. For saving an entire hierarchy, prefabs are recommended. Fluid templates will work in all versions of Fluvio, and will be backwards compatible.

Editor Features and More

Added a new Fluvio Inspector window - this aggregates all selected inspectors for Fluvio components, and provides a menu with help information and more.

Changes and Fixes

API Changes

  • Simplified the public API.
  • Greatly improved fluid serialization.
  • FluidParticles are now structs, for performance reasons. This changes the API somewhat - GetParticles/SetParticles methods need to be called, similar to Unity's particle system.
  • Merged functionality from the particle manager component into the main fluid component.
  • Rewrote system for adding/removing emitters and plugins. It is now much more stable and works correctly at runtime.
  • Added an event system for various fluid events.
  • Added a brand new broad phase API. The broad phase improves performance dramatically by using spatial partitioning. Broad phases have optimizations for both 2D and 3D.
  • Added a FluidAttractor API, similar to the emitter API
  • (Unity 4 only) All components are now in namespaces - wrapper components have been removed.
  • (Fluvio Pro only) IFluidBroadPhase interface for creating additional broad phases.
  • (Fluvio Pro only) Fluid Debug component can profile ms per frame for each stage of the solver.

Solver Changes

  • Each fluid now only has one solver, instead of a solver per particle.
  • Fixed mathematical errors in the kernel functions, most notably the viscosity kernel.
  • Fluvio now uses a different pressure constant and gradient calculation (gas constant instead of speed of sound) - this helps prevent floating point rounding errors, and is more stable.
  • Added a minimum density property for stability reasons.
  • Fixed damping, and added a velocity limit/dampening system for greatly improved stability.
  • Fluid solver integrates positions and velocities directly - no more stacking fluids!
  • Removed the adaptive solver for now - it may be reintroduced at a later date.

Other Changes

  • Exposed the underlying Shuriken particle system to the editor.
  • All components now support multi-edit.
  • Default settings now have a more reasonable scale.
  • Added editor optimization - this will decrease the amount of fluid particles while in the editor to accommodate editor overhead.
  • Emitters no longer use coroutines and emit at a set point during a fluid's frame. This allows for more performance and stability.
  • Mesh and voxel emitters now require a mesh filter.
  • Added visualization for mesh and voxel emitters.
  • Made a much better visualization for ellipsoid emitters. They can now also be resized through a scale handle.
  • Fluid destructors can now emit splash effects. This replaces the FluidDecal system.
  • Added a new default particle texture and normal map.
  • Removed many depreciated assets from the project folder.
  • Changelogs are now exported from Markdown to HTML and look much nicer ;)

Compatibility Notice

Fluvio now uses Shuriken for particle rendering and requires Unity 3.5+. The legacy particle system is no longer supported. Shuriken world collision requires Unity 4.0+. Fluvio 2.2 is not backwards compatible with older Fluvio versions - please remove older versions of Fluvio from any existing project before upgrading!
