Fluid

Overview

A fluid represents any liquid or gas as a set of simulated particles. This component is the main component for Fluvio.

See Also
Fluid

Properties

  • Editor Optimization - If enabled, reduce the amount of particles simulated while in the editor. Cannot be changed while in Play mode.
  • Editor Downsample - The amount to downsample particles by in the editor. Cannot be changed while in Play mode.
  • GPU Acceleration - GPU accelerated fluids are not yet implemented.
  • Use Gravity - If enabled, gravity will be simulated. Gravity can be changed in the physics settings.
  • Dimensions - The axes to simulate on. XYZ = 3D, XY/XZ/YZ = 2D.
  • Broad Phase Type - Fluvio uses spatial partitioning to optimize its solver. This controls the type of broad phase to use. A broad phase is recommended for all fluids with more than 30-50 particles. Cannot be set at runtime.
  • Broad Phase Size - If a broad phase is enabled, controls the size of the broad phase tree. This should be set based on the maximum distance a fluid is expected to travel. Changing this at runtime is not recommended.
  • Broad Phase Offset - If a broad phase is enabled, controls the offset of the broad phase tree. This should be set based on the maximum distance a fluid is expected to travel. Changing this at runtime is not recommended.
  • Particle Skin Width - Controls how far a particle must travel before synchronizing with the broad phase. Higher values will give better performance at the cost of accuracy/stability.
  • Smoothing Distance - The size of the smoothing kernel for SPH simulation. This controls the distance at which a particle will interact with other particles. Changing this at runtime with a broad phase is not recommended.
  • Max Interactions - The maximum amount of interactions allowed per particle per frame. Decreasing this value will give better performance, but will greatly reduce stability.
  • Particle Mass - The mass of each particle. The total mass of a fluid volume can be found by multiplying this by the particle count.
  • Initial Density - The initial density of each particle. Particles will move to fill low density areas.
  • Minimum Density - The minimum allowed density for a particle. Increase this value to help resolve stability issues.
  • Viscosity - The viscosity of the fluid. Higher values = "thicker" fluids.
  • Gas Constant - The gas constant of the fluid. The gas constant is used to calculate pressure. A higher value will produce more pressure, similar to raising the temperature. Eventually, a fluid will boil into a gas.
  • Interactive Force - This force is applied to the fluid every frame. Good for wind or secondary effects like rising gasses that are lighter than the air around it.
  • Damping - Damping factor used during physics integration.
  • Velocity Limit - Max velocity limit applied to fluid particles. Velocities above this value will be dampened. This helps with stability.
  • Velocity Dampen - The amount to dampen particle velocity by if over the limit - a value of 0.5 will reduce velocity by 50%.
  • Solver Iteration Count - How often the solver should run per frame. Higher values will use more processing power. A value of 1 or 2 is recommended.
