Collisions and World Interaction

Overview

There are two different ways that Fluvio can interact with the world - either through Shuriken collisions or rigidbody collisions.

Shuriken collisions

Shuriken collisions require the Fluid Particle Renderer component, and a linked Particle System. The Fluid Rigidbody Particles component must not be present.

To enable Shuriken collisions:

  1. Enable the collision module under Shuriken
  2. Check the "Use External Forces" box in the Fluid Particle Renderer component.
Note
Unity 3.5 only supports collision with planes - Unity 4 supports full world collision for particles. In Unity 4, this will also allow external forces like wind zones to work on particles.

Rigidbody collisions

Rigidbody collisions require the Fluid Rigidbody Particles component.

To enable rigidbody collisions:

  1. Add the Fluid Rigidbody Particles component - Component > Fluvio > Plugins > Fluid Rigidbody Particles
  2. Adjust the component properties as desired.
Note
"Use External Forces" on the Fluid Particle Renderer will be automaticaly disabled if rigidbody collisions are used.

Collision events

Rigidbody collisions will trigger normal Unity collision events. However, Shuriken collisions do not trigger any events and are not compatible with the Fluid Collision Sound plugin.
