Using Fluid Platform Profiles

Overview

Fluid platform profiles are objects that change the configuration of a fluid based on the current runtime platform. They are useful when using the same fluid in a multi-platform project. For example, the particle count can be changed for mobile targets compared to desktop targets.

Creating a profile

To create a profile, go to either Thinksquirrel > Fluvio > Create Fluid Platform Profile or use the create menu in the project folder. This will create a new profile object and select it.

Loaing profiles

To load profiles, you will need to use the Fluid Loader component. To create a Fluid Loader, go to Thinksquirrel > Fluvio > Create Fluid Loader. Drag and drop the desired profiles from the project onto this component. When the scene is loaded, each profile will be checked, in order. The first profile to match the target platform will then be loaded.

Fluid Loader

Properties

Checking for a profile

  • Enable Profile - If this is not enabled, the profile will always be skipped.
  • Always Apply Profile - If enabled, this profile will always be applied. This option is useful for fallback profiles.
  • Platform - The target platform for the fluid profile.
  • Check iPhone Generation - If enabled, the iPhone generation must match the specified value in order to enable the profile.
  • iPhone Generation - The iPhone generation to check for.
  • Check DPI - If enabled, the target DPI will be checked in order to determine whether the profile should be enabled.
  • Resolution Comparison - The comparison to use when checking the DPI (Greater Than, Less Than, or Equal).
  • DPI - The target DPI to check for.

Profile settings

  • Prefabs - All of the following prefabs will be loaded if the profile is found to be a match. A Fluid component should be present on one of the prefabs.
